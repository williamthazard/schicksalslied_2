# schicksalslied 2.0 — Sub-Plan C: Params Menu + LFOs + Soak Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build out the full params menu (~7000 params), wire LFOs across all voice/sampler/oneshot/crow params using the standard Norns LFO library, add MIDI as an 11th row-2 role, extend panic to silence crow/wtape/MIDI, and verify the full system with PSET round-trip + CPU budget + 30-min soak.

**Architecture:**
- Param IDs follow consistent naming: `cell_<x>_<y>_<param>` for per-cell, `sampler_<slot>_<param>`, `oneshot_<slot>_<param>`, `grain_<n>_<rate>` for grain LFOs, `lfo_<id>_*` (auto-generated by standard Norns LFO library).
- Helper modules in `lib/` keep `schicksalslied.lua` small: `voice_params.lua` (param-block builders), `lied_lfos.lua` (LFO instantiation tables), `midi_role.lua` (MIDI role state + dispatch).
- Bus routing requires SC-side `*_reroute` commands because `Out.ar`'s bus arg is sampled at Synth-instantiation time; live routing changes free + reallocate voice subgroups.
- LFOs use the standard `lfo` library's built-in enable param + visibility (not the spec's depth-driven scheme — verified incompatible during plan-writing; the library hides its own depth param when state=off).
- Panic extends to all crow ii devices that expose a silence verb (jf, wtape) + zero crow CV outputs + MIDI all-notes-off.

**Tech Stack:** Norns Lua (`require 'lfo'`, `require 'musicutil'`, `midi`, `crow`, `params`, `clock`), SuperCollider (Crone, SynthDef, Group, Ndef).

**Pre-conditions (from sub-plans A and B, verified on hardware):**
- All voice SC classes (`TriSin`, `Ringer`, `Sampler`, `OneShot`) implement `setParam('all', key, val)` with `.lag3` smoothing.
- Engine commands exist: `trisin_alloc/free/trigger/set_param`, `ringer_alloc/free/trigger/set_param`, `sampler_load/clear/trigger/set_param`, `oneshot_load/clear/trigger/set_param`, granular master amps, fbPatchMix setters, `silence_all_samplers`, `silence_all_oneshots`, `free_granular`, `set_beat_sec`.
- `Lied.sc` allocates buses + master FX + sets up lazy granular allocation. Granular chain has 8 grain synths with Ndef LFOs for pan/cutoff/resonance.
- `lib/sequencer.lua` owns `Seq[x][y]`, `Toggled[x][y]`, `Seq_Mode[x][y]`, `Value_Mode[x][y]`, clock loops.
- `lib/cell_roles.lua` owns `Roles.ENUM` (10 entries), per-cell `cell_role[x]`, lazy `ensure_allocated` for TriSin/Ringer.
- `schicksalslied.lua` has `init()`, `cleanup()`, `panic()` (currently JF-only), `tap_tempo()`, `Sequencer.toggle_pause()`, grid/screen handlers, `add_params()` with file params + row-8 amp controls.

```supercollider
8.do({ arg i;
    var panRate = grainPanRates[i];
    var cutoffRate = grainCutoffRates[i];
    var resRate = grainResRates[i];
    grainPanLFOs[i] = Ndef(
        ("grainPan" ++ i).asSymbol,
        { |rate=8| LFTri.kr(1 / (rate * beat_sec)).range(-1, 1); }
    );
    grainPanLFOs[i].set(\rate, panRate);
    grainCutoffLFOs[i] = Ndef(
        ("grainCutoff" ++ i).asSymbol,
        { |rate=8| LFTri.kr(1 / (rate * beat_sec)).range(500, 15000); }
    );
    grainCutoffLFOs[i].set(\rate, cutoffRate);
    grainResLFOs[i] = Ndef(
        ("grainRes" ++ i).asSymbol,
        { |rate=8| LFTri.kr(1 / (rate * beat_sec)).range(0, 2); }
    );
    grainResLFOs[i].set(\rate, resRate);
});
```

Why `|rate=8| ...` syntax: Ndef's function-arg syntax exposes the argument as a settable control. `grainPanLFOs[i].set(\rate, v)` updates the LFO period without freeing/re-allocating the Ndef.

### Task 1.5: Wire `clock_tempo` change → `engine.set_beat_sec`

**Files:**
- Modify: `schicksalslied.lua` (init: call `set_beat_sec` once; bind `clock_tempo` action)

**Rationale:** `Lied.sc` defaults `beat_sec = 0.5` (120 BPM). The granular delay buffer is sized using `beat_sec * 512` at lazy-alloc time. If the user runs at a different tempo or changes tempo mid-set, the buffer / LFO periods drift. Spec §6 calls for `engine.set_beat_sec(clock.get_beat_sec())` at init + on tempo change.

- [ ] **Step 1: Hook `set_beat_sec` into init()**

In `schicksalslied.lua`, in `init()` after `params:bang()` (around line 438), add:

```lua
    -- Push initial beat_sec to SC, then re-push on every clock_tempo change.
    engine.set_beat_sec(clock.get_beat_sec())
    params:set_action('clock_tempo', function(bpm)
        engine.set_beat_sec(clock.get_beat_sec())
    end)
```

Note: `params:set_action` replaces any existing action on `clock_tempo`. Norns's built-in clock_tempo action handles the actual tempo change internally via Lua's `clock` module; setting our own action wraps but does not need to call any prior action (Norns's clock implementation listens on the param value, not on the action chain).

**Files:**

**Rationale:** Current panic only does `crow.ii.jf.run(0)`. Spec §11 says K1 frees all running synths + does JF run(0). User feedback in plan-review session corrected scope to include w/tape stop, crow CV zero, and MIDI all-notes-off.

- [ ] **Step 1: Replace `panic()` body**

In `schicksalslied.lua`, replace the `panic()` function (around lines 383-410) with:

```lua
local function panic()
    -- Clear all toggle state so the sequencer stops dispatching
    for x = 1, 16 do
        for y = 2, 8, 2 do
            Sequencer.Toggled[x][y] = false
        end
    end
    -- Free SC voice instances (will be re-allocated lazily when user re-toggles cells)
    Roles.free_all()
    -- Clear w/tape looper running flags
    Roles.looper_running = {}
    -- Silence the persistent granular delay chain + mic passthrough
    engine.set_mic_amp(0)
    engine.set_mic_dry_amp(0)
    engine.set_granular_out_amp(0)
    engine.set_fb_amp(0)
    -- Fully free the granular chain (frees ~50-70% baseline CPU; user must
    -- re-toggle row 8 col 14/15/16 to bring it back)
    engine.free_granular()
    -- Hard-stop in-flight sampler and one-shot playback
    engine.silence_all_samplers()
    engine.silence_all_oneshots()
    -- Stop Just Friends run mode
    crow.ii.jf.run(0)
    -- Stop w/tape playback explicitly
    crow.ii.wtape.play(0)
    -- Zero crow CV outputs (1..4) to halt any in-flight envelope-driven CVs.
    -- AR envelopes already release naturally; this guarantees the final value.
    for n = 1, 4 do
        crow.output[n].volts = 0
    end
    -- MIDI: send All-Notes-Off (CC 123) on every channel of the active device.
    -- Midi_Role.device is set by midi_role.lua's init when a device is configured;
    -- nil-safe so panic doesn't break on systems without MIDI configured.
    if Midi_Role and Midi_Role.device then
        for ch = 1, 16 do
            Midi_Role.device:cc(123, 0, ch)
        end
    end
    -- Note: w/syn and w/del don't expose direct silence verbs. Their voices
    -- decay naturally via internal envelopes. Clearing Toggled (above) is
    -- the main mitigation — no new triggers will reach them.
    Grid_Dirty = true
    print('PANIC: silenced everything')
end
```

```lua
-- lib/midi_role.lua — MIDI output role state + dispatch
-- Owns: the connected MIDI device handle, per-cell active-note tracking.
-- Pattern modeled on tehn/awake.lua: dynamic vports list, per-note tracking
-- for clean note-offs, no all-channel CC123 blast.

local Midi = {}

-- The connected MIDI device (set by Midi.init / device-change action).
-- nil-safe so other modules (panic, dispatch) can check before calling.
Midi.device = nil

-- Per-cell active-note tracking. Keyed by cell_id ('1_2'); value is
-- { note=N, channel=C } for the currently-sounding note from that cell.
-- We use this both to note-off the previous note before retriggering, and
-- to drive panic's all_notes_off without resorting to a CC123 blast that
-- could step on other scripts sharing the device.
Midi.active_notes = {}

-- Build the device list from midi.vports. Called once at Midi.init().
-- Each vport may have a name like "norns" (built-in) or "OP-1 field" (USB);
-- empty vports get a "port N" placeholder so the option list always has
-- 4 entries (matching Norns's vport count).
function Midi.build_device_list()
    local devices = {}
    for i = 1, #midi.vports do
        local n = midi.vports[i].name
        table.insert(devices, n ~= "none" and n or ('port ' .. i))
    end
    return devices
end

-- Initialize MIDI: connect to the device at the current midi_device param.
-- Called from schicksalslied.lua's init() AFTER add_params + params:bang.
function Midi.init()
    Midi.connect_device(params:get('midi_device') or 1)
end

-- (Re-)connect to the MIDI device at vport index n.
-- Called from the midi_device param action.
function Midi.connect_device(n)
    Midi.device = midi.connect(n)
    print(string.format('MIDI: connected vport %d (%s)',
        n, Midi.device.name or 'unknown'))
end

-- Dispatch a MIDI note event for the cell at (x, y).
-- Called from cell_roles.lua's MIDI role dispatcher.
-- seq is the cell's sequins-byte-reader function.
function Midi.dispatch(x, y, seq)
    if Midi.device == nil then return end
    local cell_id = string.format("%d_%d", x, y)
    local channel = params:get('cell_' .. x .. '_2_midi_channel') or 1
    local note = seq() % 32 + 49  -- MIDI note range 49..80 (C3..G♯5)
    local vel = math.min(127, seq() % 32 + 49)  -- velocity range 49..80
    -- Note-off any previously-active note from this cell. Use the PREVIOUS
    -- channel (which may differ from current if channel changed mid-sequence)
    -- so the off lands on the same channel the on was sent to.
    -- Note: Norns MIDI signature is dev:note_off(note, vel, ch); vel may be nil.
    local prev = Midi.active_notes[cell_id]
    if prev then
        Midi.device:note_off(prev.note, nil, prev.channel)
    end
    Midi.active_notes[cell_id] = { note = note, channel = channel }
    Midi.device:note_on(note, vel, channel)
    -- Schedule note-off after gate time. Capture note + channel in the
    -- closure so the off always targets the on's exact pair, even if a
    -- subsequent dispatch replaces active_notes[cell_id].
    -- Gate time is a single global param (modeled on awake's simpler
    -- "one gate control" approach — per-cell gate was over-engineered).
    local gate_time = params:get('midi_gate_time') or 0.1
    clock.run(function()
        clock.sleep(gate_time)
        Midi.device:note_off(note, nil, channel)
        -- Only clear active_notes if it's still our pair (a newer dispatch
        -- may have already replaced it; that dispatch's note-off-of-previous
        -- already cleaned up our on).
        local current = Midi.active_notes[cell_id]
        if current and current.note == note and current.channel == channel then
            Midi.active_notes[cell_id] = nil
        end
    end)
end

-- All-notes-off — iterate tracked active_notes and send a note_off for each.
-- Cleaner than blasting CC 123 on all 16 channels (which would interfere
-- with other scripts/processes sharing the device).
function Midi.all_notes_off()
    if Midi.device == nil then return end
    for _, entry in pairs(Midi.active_notes) do
        Midi.device:note_off(entry.note, nil, entry.channel)
    end
    Midi.active_notes = {}
end

-- Channel-change handler. When a cell's MIDI channel changes mid-sequence,
-- note-off the cell's currently-active note (on the OLD channel) so it
-- doesn't get stranded. Called from the per-cell midi_channel param action.
function Midi.on_channel_change(x, y)
    local cell_id = string.format("%d_%d", x, y)
    local entry = Midi.active_notes[cell_id]
    if entry and Midi.device then
        Midi.device:note_off(entry.note, nil, entry.channel)
        Midi.active_notes[cell_id] = nil
    end
end

return Midi
```

```lua
local VoiceParams = {}

-- Bus routing enum + sentinel values. The bus index passed to SC depends on
-- the Lied kernel's bus allocation (dryBus, reverbBus, delayBus); to keep
-- Lua-side semantics stable across kernel changes, we use names here and
-- translate at action time.
VoiceParams.BUS_ROUTING_OPTIONS = { 'dry', 'reverb', 'delay+reverb' }

-- Resolve a bus_routing string to the actual SC audio bus index.
-- The Lied kernel exposes its buses via its `dryBus`, `reverbBus`, `delayBus`
-- vars; in Lua we know the indices via engine commands. For now, we hardcode
-- the expected indices: dryBus = 0, reverbBus = 2, delayBus = 4 (stereo, so
-- each bus consumes 2 channels). If Lied's bus allocation changes, update here.
-- Actually we use named engine commands that accept a routing key, so the
-- mapping lives in Lied.sc — but for cell-level params the simpler approach
-- is to pass the routing name through directly.
-- For this implementation: use engine command per-routing.
function VoiceParams._dispatch_bus(routing, dispatch_fn)
    -- routing is a 1-3 index into BUS_ROUTING_OPTIONS.
    -- dispatch_fn is called with the bus name; the engine command's behavior
    -- selects the right bus.
    local name = VoiceParams.BUS_ROUTING_OPTIONS[routing]
    dispatch_fn(name)
end

        type = 'control',
        id = 'sampler_' .. slot .. '_cutoff',
        name = 'sampler ' .. slot .. ' cutoff',
        controlspec = controlspec.new(20, 18000, 'exp', 1, 12000, 'Hz'),
        action = function(v) engine.sampler_set_param(slot, 'cutoff', v) end,
    }
    params:add{

        id = 'sampler_' .. slot .. '_resonance',

    }
    params:add{
        type = 'number',
        id = 'sampler_' .. slot .. '_polyphony',
        name = 'sampler ' .. slot .. ' polyphony',
        min = 1, max = 8, default = 1,
        -- Polyphony param stored in cell_roles.polyphony table; action sets it
        action = function(v)
            local Roles = require 'lib/cell_roles'
            -- Sampler poly is shared across its trigger and rate cell pair;
            -- spec §5 says default 1, configurable 1-8. Sub-plan B's Roles
            -- table stores per-cell poly; we set it for both trigger cells

-- Map bus_routing index (1..3) to the actual SC bus number for the Lied kernel.
-- The kernel allocates buses in this order during init:
--   dryBus    = stereo bus, starts at index 0 (channels 0-1)
--   reverbBus = stereo bus, next (channels 2-3)
--   delayBus  = stereo bus, next (channels 4-5)
-- But because of how Norns/Crone allocate the master output bus (channels 0-1
-- typically), the Lied kernel uses Bus.audio(server, 2) — which returns
-- post-output-bus indices. The exact numbers are server.options.numOutputBusChannels
-- dependent. We use a small lookup: 1→dry (route to dryBus.index),
-- 2→reverb (route to reverbBus.index), 3→delay+reverb (route to delayBus.index).
-- Since the Lua side can't directly query SC bus indices, we hardcode the
-- known triplet for Lied.sc: dryBus.index = 4, reverbBus.index = 6, delayBus.index = 8
-- (these depend on Norns's 2-channel output config + Lied's allocation order).
-- VERIFY ON HARDWARE — if bus routing has no audible effect, print these from SC.
function VoiceParams.bus_idx_for(routing_idx)
    -- Routing 1 = dry, 2 = reverb (post-delay), 3 = delay+reverb.
    if routing_idx == 1 then return 4    -- dryBus (post-output)
    elseif routing_idx == 2 then return 6  -- reverbBus
    elseif routing_idx == 3 then return 8  -- delayBus
    end
    return 4  -- default dry
end

    params:set('sampler_' .. slot .. '_resonance',  math.random(50, 300) / 100)
    params:set('sampler_' .. slot .. '_pan',        (math.random() * 2) - 1)
end
```

```lua
-- lib/lied_lfos.lua — LFO instantiation for every voice/sampler/oneshot/crow
-- target param. Uses the standard Norns LFO library (require 'lfo').
--
-- The standard library:
--   - LFO:add{ ... } returns an LFO object and adds ~13 params under
--     IDs like lfo_<id>, lfo_shape_<id>, lfo_depth_<id>, etc.
--   - The library handles its own visibility: lfo_<id> param (off/on)
--     auto-hides/shows the other params via params:hide/show.
--   - lfo:start() and lfo:stop() are called automatically by the library
--     when lfo_<id> changes; no wrapper action needed.
--
-- We just need to:
--   1. Pre-build the LFO with a meaningful id, default depth=0,
--      and an action function that calls params:set on the target param.
--   2. Add it to the params menu via lfo:add{...}.
--   3. NEVER start the LFO at init — depth=0 + state=off means no overhead.
--
-- Spec §10 inline correction: the original spec proposed depth-driven implicit
-- start/stop. This is incompatible with the standard library which hides the
-- depth param when state=off (user couldn't re-enable). Riding the library
-- is simpler and matches every other Norns script's LFO UX.

local LFO = require 'lfo'
local LiedLfos = {}

-- Storage for the LFO objects (so they survive past init scope, for inspection)
LiedLfos.bound = {}

-- Bind an LFO that, when active, drives `target_param_id`.
-- Each LFO call adds 13 params (state, shape, depth, phase, offset, min, max,
-- baseline, mode, clocked rate, free rate, reset, reset_target) — auto-named
-- via lfo_<lfo_id>_* pattern.
function LiedLfos.bind(lfo_id, target_param_id, min, max)
    local lfo = LFO:add{
        shape  = 'sine',
        min    = min or 0,
        max    = max or 1,
        depth  = 0,
        mode   = 'clocked',
        period = 4,
        action = function(scaled, raw)
            params:set(target_param_id, scaled)
        end,
    }
    LiedLfos.bound[lfo_id] = lfo
    return lfo
end

-- Convenience: bind a batch of LFOs from a config table.
-- config = { { lfo_id, target_param, min, max }, ... }
function LiedLfos.bind_batch(config)
    for _, entry in ipairs(config) do
        LiedLfos.bind(entry[1], entry[2], entry[3], entry[4])
    end
end

return LiedLfos
```

```
### Architecture (uses standard Norns LFO library directly)

The standard `lfo` library handles its own enable/disable + visibility:

- Adding an LFO via `LFO:add{...}` exposes a `lfo_<id>` option param (off/on)
  which serves as the master switch.
- When the LFO is "off", the library calls `lfo_params_visibility("hide", id)`
  internally, hiding all other LFO params (depth, shape, period, etc.).
- When set to "on", the library calls `:start()` automatically and shows
  the sub-params.

The original Section-10 proposal to make depth itself the on/off driver
was incompatible with this library design: depth cannot be raised above 0
to enable the LFO because depth is hidden while the LFO is off. Sub-plan C
adopts the library's built-in enable mechanism directly.

This adds one param per LFO vs the original proposal (~332 LFOs ≈ +332 params)
but is simpler and matches every other Norns script's LFO UX.
```
